Testing: ComposeStrips — assemble strips into a flush panel

ComposeStrips is the starting operation of every design: it takes an ordered list of StripDef (species + width) plus a global thickness and length, and builds a flush panel by placing each strip against the next along +X. This harness is the first to expose interactive Input and Operation tiles — the other harnesses (Cut, Arrange, Trim) are URL-only.

Expected in each tile:

  • Input (interactive) — the raw inventory of strips. Add / remove, change species + width, set global thickness and length. Order here is only insertion order, not the arrangement order downstream; that lives in the Operation tile.
  • Operation (interactive) — drag strips to reorder them. Rendered in the same top-down SVG vocabulary as the Arrange operation tile — species-coloured polygons, muted strokes, real mm-scaled widths and lengths — but with small gaps between strips so the pieces read as "not yet flush" and narrow strips stay grabbable. Flush assembly happens downstream in the Output tile.
  • Output — the composed panel in 3D. Free orbit, home button, XYZ axis gizmo in the upper-right.

URL params seed the initial state only. Edits in the Input and Operation tiles do NOT write back to the URL. Refresh = return to URL state. See the sidebar for the param reference.
